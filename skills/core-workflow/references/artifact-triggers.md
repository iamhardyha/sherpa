# 산출물 트리거 결정 가이드 — 언제 무엇을 만드나

§2 5종 표의 실행 가이드. "지금 무슨 산출물을 건드려야 하나"를 빠르게 판정한다. 정본은 스펙 §2.

## 시간축으로 고른다

| 순간 | 질문 | 산출물 |
|---|---|---|
| 결정할 때 | *왜* 이렇게 정했나 | **adr** |
| 작업 전 | 앞으로 *무엇을* 할 건가 | **plan** |
| 상시 | 지금 *어떻게* 동작하나 | **spec** |
| 작업 후 | 실제로 *무엇을* 했나 | **report** |
| 은퇴 | 더는 현역 아님 | **archive** |

## 결정 흐름

```
변경이 트리비얼? ── 예 → 판단층·트리거 의무 면제 (바닥만 — §6-D)
      │ 아니오
      ▼
비가역·대안 있는 결정을 내렸나? ── 예 → ADR (§2.3, supersede면 양방향 링크)
      │
      ▼
코드가 spec의 refs를 바꿨나? ── 예 → 해당 spec 갱신 (안 하면 드리프트 실패 §6-I)
      │
      ▼
plan 태스크를 전부 끝냈나? ── 예 → report 작성 (plan 역참조 → done 자동 파생 §2.2)
      │
      ▼
은퇴한 산출물이 있나? ── 예 → /sherpa-sweep 후보 (사람 승인분만 archive)
```

## 각 산출물 트리거 기준

### adr — 결정할 때
- **트리거**: 비가역이거나 대안이 있던 설계 결정.
- 작업 중 의도 피벗은 plan `## 변경 이력`이 정본, *그 전환이 비가역·대안 결정일 때만* ADR로 승격(§2.2).
- 번복은 옛 ADR 수정이 아니라 **새 ADR + 양방향 supersede**(§2.3).

### plan — 작업 전
- **트리거**: 트리비얼 아닌 다중 단계 작업. status `active`, `## 작업` 체크박스.
- 설계 필요면 `## 설계`(브레인스토밍). 기준은 SKILL.md 1단계.

### spec — 상시 (드리프트가 강제)
- **트리거**: 코드가 spec의 `refs`(경로·심볼·글롭)를 바꿈 → 해당 spec 갱신.
- 통과 시 `last-verified`를 오늘로. **stale ≠ retire**(§2.7): 오래됨=재검증(살아있음, archive 아님), refs 전멸/superseded-by=retire 후보.

### report — 작업 후 (plan 완료가 신호)
- **트리거**: plan 태스크 전부 체크 → report 작성, 상단에 `plan: {slug}` 역참조.
- 코드 변경 자체가 트리거가 아님 — *계획 종결*이 트리거(§6-A).
- 한 일·검증·결정·이탈 4섹션.

### archive — 은퇴 (유예 후 sweep)
- **트리거**: `/sherpa-sweep`이 나이 지난 done·abandoned·superseded plan, superseded ADR, retire 후보 spec, 타입별 최신 N 초과 lens를 후보 제안 → **사람 승인분만** 이동.
- `active`·`paused`는 절대 archive 안 함. done은 재개될 수 있어 *즉시* archive 안 함(§2.2).

## lens는 트리거가 아님

lens는 5종과 직교(시간축 밖, §2.4). 사람이 "길 잃었을 때 들여다보는" 도구라 `/sherpa-lens`로 명시 호출. 라이브 뷰는 쌓지 말고 재생성(수명 티어 §2.4).
